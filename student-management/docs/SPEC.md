# 学生管理系统 MVP - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
学生管理系统 (Student Management System)

### 1.2 项目类型
Web 应用系统 - 最小可行产品 (MVP)

### 1.3 项目目标
构建一个完整的学生信息管理系统，实现学生数据的增删改查 (CRUD) 操作，采用现代化的前后端分离架构。

### 1.4 技术栈
- **后端**: Laravel 12.x (PHP 8.3)
- **前端**: Vue 3 + Vite
- **数据库**: MySQL 8.0 (Docker 容器)
- **Web服务器**: Nginx
- **开发环境**: WSL

---

## 2. 功能需求

### 2.1 核心功能列表

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|-------|
| 学生列表 | 显示所有学生信息，支持分页 | P0 |
| 学生详情 | 查看单个学生的详细信息 | P0 |
| 添加学生 | 创建新的学生记录 | P0 |
| 编辑学生 | 修改现有学生信息 | P0 |
| 删除学生 | 删除学生记录 | P0 |
| 搜索学生 | 按姓名/学号搜索 | P1 |
| 数据验证 | 前后端数据校验 | P1 |

### 2.2 用户故事

#### US001: 查看学生列表
作为管理员，我希望能够查看所有学生的列表，以便了解学生概况。
- 验收标准：
  - 显示学生姓名、学号、班级、联系方式
  - 支持分页显示（每页10条）
  - 显示总学生数

#### US002: 添加学生
作为管理员，我希望能够添加新学生信息。
- 验收标准：
  - 必填字段：姓名、学号、性别、出生日期、班级
  - 选填字段：联系电话、邮箱、地址
  - 学号唯一性校验
  - 提交后自动刷新列表

#### US003: 编辑学生
作为管理员，我希望能够修改学生信息。
- 验收标准：
  - 预填充现有数据
  - 支持修改所有字段
  - 学号唯一性校验（排除自身）
  - 保存后自动刷新列表

#### US004: 删除学生
作为管理员，我希望能够删除学生记录。
- 验收标准：
  - 删除前二次确认
  - 删除后自动刷新列表
  - 显示成功/失败提示

#### US005: 搜索学生
作为管理员，我希望能够快速搜索学生。
- 验收标准：
  - 支持按姓名模糊搜索
  - 支持按学号精确搜索
  - 实时显示搜索结果

## 3. 数据模型

### 3.1 学生表 (students)

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|-------|------|------|------|------|
| id | bigint | - | PK, AUTO_INCREMENT | 主键 |
| student_no | varchar | 20 | UNIQUE, NOT NULL | 学号 |
| name | varchar | 50 | NOT NULL | 姓名 |
| gender | enum | - | NOT NULL | 性别 |
| birth_date | date | - | NOT NULL | 出生日期 |
| class_name | varchar | 50 | NOT NULL | 班级 |
| phone | varchar | 20 | NULL | 联系电话 |
| email | varchar | 100 | NULL | 邮箱 |
| address | varchar | 255 | NULL | 地址 |
| created_at | timestamp | - | - | 创建时间 |
| updated_at | timestamp | - | - | 更新时间 |

### 3.2 数据字典

#### gender 枚举值
- `male` - 男
- `female` - 女
- `other` - 其他

## 4. 接口设计

### 4.1 API 端点

| 方法 | 路径 | 描述 |
|-----|------|------|
| GET | /api/students | 获取学生列表 |
| GET | /api/students/{id} | 获取学生详情 |
| POST | /api/students | 创建学生 |
| PUT | /api/students/{id} | 更新学生 |
| DELETE | /api/students/{id} | 删除学生 |
| GET | /api/students/search | 搜索学生 |

### 4.2 响应格式

#### 成功响应
```json
{
  "success": true,
  "data": {},
  "message": "操作成功"
}
```

#### 错误响应
```json
{
  "success": false,
  "data": null,
  "message": "错误信息",
  "errors": {}
}
```

### 4.3 详细接口规格

#### GET /api/students
- Query Parameters:
  - `page` (optional): 页码，默认1
  - `per_page` (optional): 每页数量，默认10
- Response: 学生列表 + 分页信息

#### POST /api/students
- Request Body:
```json
{
  "student_no": "2024001",
  "name": "张三",
  "gender": "male",
  "birth_date": "2005-03-15",
  "class_name": "高三(1)班",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "address": "北京市朝阳区"
}
```

## 5. 验收标准

### 5.1 功能验收

| 编号 | 测试场景 | 预期结果 |
|-----|---------|---------|
| TC001 | 访问学生列表 | 显示学生列表，分页正常 |
| TC002 | 添加有效学生 | 学生添加成功，列表刷新 |
| TC003 | 添加无效数据 | 显示相应错误提示 |
| TC004 | 编辑学生信息 | 信息更新成功 |
| TC005 | 删除学生 | 学生删除成功 |
| TC006 | 搜索学生 | 显示匹配的搜索结果 |

### 5.2 界面验收

- 界面加载无白屏
- 响应式布局，支持移动端
- 操作反馈明确（加载中、成功、失败）
- 表单验证提示清晰

### 5.3 性能要求

- 页面首次加载 < 3秒
- API 响应时间 < 500ms
- 支持 1000+ 学生数据
