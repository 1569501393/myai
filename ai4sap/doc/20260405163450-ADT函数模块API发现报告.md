## 二、API 测试结果

### 2.1 测试的端点

| 端点 | 状态 | 错误信息 |
|------|------|----------|
| `/sap/bc/adt/functions/groups` | ❌ | Parameter `groupname` could not be found |
| `/sap/bc/adt/functions/groups?groupname=SAPLCOBK` | ❌ | Parameter `groupname` could not be found |
| `/sap/bc/adt/functions/groups/SAPLCOBK/fmodules` | ❌ | URI-Mapping cannot be performed |
| `/sap/bc/adt/functions/groups/ZFGDATAGEN1/fmodules` | ❌ | URI-Mapping cannot be performed |
| `/sap/bc/adt/functions/groups/SAPLCOBK/includes` | ❌ | Parameter `fugrincludename` could not be found |
| `/sap/bc/adt/functions/validation` | ❌ | Resource controller does not support method GET |

### 2.2 错误分析

所有函数相关端点都返回错误，常见错误类型：

1. **ExceptionParameterNotFound**: 参数找不到
   - `groupname` 参数未找到
   - `fugrincludename` 参数未找到

2. **uriMappingError**: URI 映射错误
   - `/sap/bc/adt/functions/groups/{name}/fmodules` 无法映射

3. **ExceptionResourceNotFound**: 资源不存在
   - `/IWFND/CL_TRANSACTION_HANDL_GC` 对象类型不存在

### 2.3 根本原因

SAP Gateway 服务 `/IWFND/CL_TRANSACTION_HANDL_GC` 未正确配置或激活。

## 三、函数模块信息获取替代方案

### 3.1 通过 TFDIR 表查询

使用数据预览端点查询函数模块信息：

```bash
# 获取 CSRF Token
curl -c cookies.txt -u "U1170:hysoft888999" \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/discovery" \
  -H "x-csrf-token: Fetch" -D headers.txt -k > /dev/null

CSRF=$(grep -i "^x-csrf-token" headers.txt | cut -d' ' -f2 | tr -d '\r')

# 查询函数模块列表
curl -b cookies.txt \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/datapreview/freestyle?rowNumber=10" \
  -H "Content-Type: text/plain" \
  -H "X-CSRF-Token: $CSRF" \
  -d "SELECT funcname, pname, include FROM tfdir WHERE funcname LIKE 'Z%'" -k
```

### 3.2 TFDIR 表字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `FUNCNAME` | CHAR 30 | 函数模块名称 |
| `PNAME` | CHAR 40 | 函数组主程序 (Program Pool Name) |
| `INCLUDE` | CHAR 4 | Include 编号 |
| `FREEDATE` | DATS | 释放日期 |
| `APPL` | CHAR 1 | 应用组件 |
| `MAND` | CHAR 1 | 强制标志 |
| `FMODE` | CHAR 1 | 功能模式 |
| `HOST` | CHAR 32 | 宿主系统 |
| `UTASK` | CHAR 1 | 用户任务 |
| `PNAME_MAIN` | CHAR 40 | 主程序名称 |
| `RFCSCOPE` | CHAR 1 | RFC 范围 |
| `RFCVERS` | CHAR 1 | RFC 版本 |

## 五、函数模块源码获取

### 5.1 通过程序获取

函数模块源码存储在对应的程序 Include 中。根据 TFDIR-PNAME：

| Include 类型 | 命名规则 | 说明 |
|-------------|----------|------|
| 主程序 | `SAPL{group}` | 函数组主程序 |
| 函数模块 | `L{include_num}{group}` | 各个函数模块 |
| TOP Include | `L{group}TOP` | 全局数据声明 |

### 5.2 获取函数组主程序

```bash
# 获取函数组主程序源码
curl -b cookies.txt -u "U1170:hysoft888999" \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/programs/programs/SAPLZFGDATAGEN1/source/main"
```

  "http://mysap.goodsap.cn:50400/sap/bc/adt/programs/includes/L07SAPLZFGDATAGEN1"

## 六、函数模块相关 ADT URI 模板

### 6.1 完整的 URI 模板

| 模板 | 说明 |
|------|------|
| `/sap/bc/adt/functions/groups/{groupname}` | 函数组元数据 |
| `/sap/bc/adt/functions/groups/{groupname}/fmodules` | 函数模块列表 |
| `/sap/bc/adt/functions/groups/{groupname}/includes` | 函数组 Includes |
| `/sap/bc/adt/functions/groups/{groupname}/fmodules/{funcname}/source/main` | 函数模块源码 |
| `/sap/bc/adt/functions/validation` | 函数组验证 (POST) |

### 6.2 URL 编码规则

函数组名称中的 `/` 需要编码为 `%2f`：

| 原名称 | 编码后 |
|--------|--------|
| `/SAPL/COBK` | `%2fSAPL%2fCOBK` |
| `/FUNCTION/GROUP` | `%2fFUNCTION%2fGROUP` |

## 八、总结

### 8.1 可用的 API

| API | 状态 | 说明 |
|-----|------|------|
| `/sap/bc/adt/datapreview/freestyle` | ✅ 可用 | SQL 查询 TFDIR 等表 |
| `/sap/bc/adt/programs/programs/{name}/source/main` | ✅ 可用 | 获取函数组主程序源码 |
| `/sap/bc/adt/programs/includes/{name}` | ✅ 可用 | 获取 Include 源码 |
| `/sap/bc/adt/functions/groups/{groupname}/fmodules` | ❌ 不可用 | 服务未配置 |

### 8.2 替代方案

由于函数组专用 API 不可用，建议使用以下替代方案：

1. **查询 TFDIR 获取函数模块列表**
2. **使用程序源码 API 获取函数组源码**
3. **通过 Include 名称获取单个函数模块源码**

### 8.3 建议

联系 SAP 管理员激活以下服务：
- `/sap/bc/adt/functions/*`
- `/IWFND/CL_TRANSACTION_HANDL_GC` Gateway 组件
