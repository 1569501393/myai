# 使用自定义端点 /sap/bc/zzjq 获取 SAP 表数据指南

## 概述

本文档说明如何使用 SAP 系统中已开启的自定义 REST API 端点 `/sap/bc/zzjq` 通过参数 `table` 获取表数据，例如获取 MARA 表的前 3 条数据。

## 已知可用端点

根据系统测试，以下端点可用：

- **端点**：`http://mysap.goodsap.cn:50400/sap/bc/zzjq`
- **方法**：GET
- **参数**：
  - `sap-client`: SAP 客户端号 (例如: 400)
  - `table`: 目标表名 (例如: mara)
  - 可选：`rows`: 返回的行数 (如果服务支持，否则服务可能忽略或使用默认值)

### 示例请求
```
http://mysap.goodsap.cn:50400/sap/bc/zzjq?sap-client=400&table=mara
```

该请求返回 MARA 表的数据（格式为 JSON）。

## 在 Python 脚本中使用

我们已经在 `sap_connector.py` 中添加了 `get_table_data` 方法来封装此调用。

### 方法签名
```python
def get_table_data(self, table_name, rows=10):
    """
    Fetch table data via custom endpoint: /sap/bc/zzjq?sap-client=400&table={table_name}
```

### 使用示例
```python
from sap_connector import SAPConnector, load_config

config = load_config()
conn = SAPConnector(config)

# 获取 MARA 表前 3 条数据
result = conn.get_table_data("MARA", rows=3)

if "data" in result:

    print(json.dumps(result["data"], indent=2, ensure_ascii=False))
```


## 最佳实践

1. **验证响应**：始终检查返回的状态码和数据格式。
2. **错误处理**：处理可能的错误，如认证失败（401）、端点或表未找到（404）、服务器错误（500）等。
3. **数据量控制**：虽然我们的示例中使用了 `rows` 参数，但服务可能忽略该参数。如果服务返回大量数据，请考虑在客户端后处理进行过滤或分页，或者咨询 SAP 管理员是否可以修改服务以支持 `rows` 参数。
4. **安全**：确保使用 HTTPS（如果可用）以保护凭据和数据传输。在生产环境中，应避免在明文中传输凭据。
5. **缓存**：如果频繁访问同一表数据，考虑缓存结果以减少对 SAP 系统的负载。
6. **日志和监控**：记录 API 调用和响应时间，以便进行性能监控和故障排除。

## 注意事项

- 该端点 `/sap/bc/zzjq` 是一个自定义开发的服务，非 SAP 标准端点。其行为和可用性取决于具体的实现。
- 在不同的 SAP 系统中，该端点可能有不同的路径或参数。请根据实际情况调整。
- 如果需要获取其他表的数据，只需修改 `table` 参数的值即可。
- 该端点可能不支持复杂的查询选项（如 `$filter`, `$select` 等），如果需要更高级的查询功能，考虑使用标准 OData 服务或创建自定义 OData 服务。

## 在 sap_connector.py 中的实现

在 `sap_connector.py` 中，我们添加了以下方法：

```python
def get_table_data(self, table_name, rows=10):
    """
    Fetch table data via custom endpoint: /sap/bc/zzjq?sap-client=400&table={table_name}
    Returns JSON data.
    """
    path = f"/sap/bc/zzjq"
    params = {
        "sap-client": self.client,
        "table": table_name
    }
    if rows:
        params["rows"] = rows

    response = self._request("GET", path, headers=self._get_headers("application/json"), params=params)

    if response.status_code == 200:
        try:
            return {"data": response.json()}
        except json.JSONDecodeError:
            return {"data": response.text, "note": "Response is not valid JSON"}
    elif response.status_code == 401:
        return {"error": "Authentication failed", "status": 401}
    elif response.status_code == 404:
        return {"error": f"Endpoint or table {table_name} not found", "status": 404}
    else:
        return {"error": f"HTTP {response.status_code}", "detail": response.text}
```
