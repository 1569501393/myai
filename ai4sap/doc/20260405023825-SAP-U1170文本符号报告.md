## 一、概述

### 1.1 数据说明

### 1.2 查询方法

通过 SAP ADT REST API freestyle 端点查询 D020T 表：

```bash
# 获取 CSRF Token
curl -c cookies.txt -u "U1170:密码" \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/discovery" \
  -H "x-csrf-token: Fetch" -D headers.txt -k > /dev/null

# 查询文本符号
CSRF=$(grep -i "^x-csrf-token" headers.txt | cut -d' ' -f2 | tr -d '\r')
curl -b cookies.txt \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/datapreview/freestyle?rowNumber=1000" \
  -H "Content-Type: text/plain" \
  -H "X-CSRF-Token: $CSRF" \
  -d "SELECT prog, dynr, dtxt FROM d020t WHERE prog LIKE 'ZMM%'"
```

## 四、说明

### 4.1 文本符号类型

D020T 表存储的文本符号类型：
1. **选择屏幕描述** - SELECT-SCREEN BEGIN OF BLOCK 等的标题
2. **参数文本** - PARAMETERS 关键字后的文本标签
3. **屏幕标题** - 自定义屏幕的标题文本

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| PROG | C | 程序名称 |
| DYNR | N | 屏幕号 |
| LANG | C | 语言代码 |
| DTXT | C | 文本描述 |

### 4.3 INFO格式说明

`SEL_SCREEN 1000 INFO:xxxxxxxxxxxx` 格式表示：
- 程序的标准选择屏幕文本
- 后面的数字是屏幕对象的内部标识

### 4.4 注意事项

- D020T 只存储屏幕级别的文本符号
- 参数级别的文本符号存储在程序源代码中
- 如需获取完整的参数文本，需要解析程序源代码
