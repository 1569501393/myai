# 动态表查询 ABAP 程序优化文档

## 概述

优化了原有的 `data perf.abap` 程序，将其转换为功能完整的 ABAP 类，支持通过 REST API 动态查询任意 SAP 表数据。

## 功能特性

### 支持的参数

| 参数 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `table` | ✅ | 表名 | `table=SPFLI` |
| `fields` | ❌ | 字段列表，逗号分隔 | `fields=CARRID,CONNID,CITYFROM` |
| `where` | ❌ | WHERE 条件 | `where=CARRID='AA'` |
| `orderby` | ❌ | 排序字段 | `orderby=CITYFROM` |
| `limit` | ❌ | 返回行数，默认 10，最大 1000 | `limit=100` |
| `offset` | ❌ | 偏移量 | `offset=10` |

### 使用示例

```bash
# 查询 SPFLI 表前 10 条数据
curl "http://sap-server:port/sap/bc/rest/z_dynamic_query?table=SPFLI"

# 指定字段
curl "http://sap-server:port/sap/bc/rest/z_dynamic_query?table=SPFLI&fields=CARRID,CONNID,CITYFROM,CITYTO"

# 带 WHERE 条件
curl "http://sap-server:port/sap/bc/rest/z_dynamic_query?table=SPFLI&where=CARRID='AA'"

# 带排序和限制
curl "http://sap.server:port/sap/bc/rest/z_dynamic_query?table=SPFLI&orderby=CITYFROM&limit=5"

# 完整示例
curl "http://sap-server:port/sap/bc/rest/z_dynamic_query?table=SPFLI&fields=CARRID,CONNID,CITYFROM,CITYTO&where=CARRID='AA'&orderby=CITYFROM&limit=10"
```

## 安全验证

### 表名验证 (`VALIDATE_TABLE_NAME`)
- 检查表名长度 (1-16 字符)
- 检查首字符必须为字母
- 检查只包含合法字符 (A-Z, 0-9, _)
- **验证表在 SAP DDIC 中存在** (查询 DD02L)

### 字段验证 (`VALIDATE_FIELDS`)
- 分割字段列表
- 验证每个字段在表中是否存在 (查询 DD03L)

### WHERE 条件验证 (`PARSE_WHERE_CLAUSE`)
- 基础 SQL 注入检测
- 过滤危险关键字 (DROP, EXEC, INSERT, UPDATE, DELETE, UNION, SELECT)

### 输入过滤 (`SANITIZE_INPUT`)
- 移除 XSS 风险字符 (<, >, &)

## 类结构

```
zcl_dynamic_query
├── if_http_extension~handle_request    # 主入口
├── validate_table_name                  # 表名验证
├── validate_fields                      # 字段验证
├── parse_where_clause                   # WHERE 条件解析
└── sanitize_input                      # 输入过滤
```

## 响应格式

### 成功响应
```json
{
  "success": true,
  "count": 10,
  "data": [
    { "carrid": "AA", "connid": "001", ... },
    ...
  ]
}
```

### 错误响应
```json
{
  "code": "INVALID_TABLE",
  "message": "Table ZTJQ9999 does not exist in SAP DDIC"
}
```

## HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 500 | 服务器内部错误 |

## 部署步骤

1. 在 SE80 创建类 `ZCL_DYNAMIC_QUERY`
2. 在 SICF 创建服务 `/sap/bc/rest/z_dynamic_query`
3. 将处理类设为 `zcl_dynamic_query`
4. 配置权限 (S_ICF, S_ABAP)
