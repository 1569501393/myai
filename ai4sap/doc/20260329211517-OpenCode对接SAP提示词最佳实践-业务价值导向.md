## 业务场景与提示词模板

### 场景 1: 业务数据分析

**业务痛点**: 业务人员需要从 SAP 表中获取数据进行分析，但依赖技术人员

**价值**: 业务人员自助获取数据，减少 IT 依赖

#### ❌ 低质量提示词
```
获取 MARA 表的数据
```

#### ✅ 高质量提示词模板

```markdown
我需要分析 SAP 物料主数据，请帮我：

1. **获取 MARA 表的前 10 条数据**，包含字段：MATNR, MTART, MBRSH, STATU, ERSDA, LAEDA
2. **分析表结构**，识别关键字段的数据类型和含义
3. **解释业务含义**：这些字段在物料管理中代表什么？
4. **输出格式**：JSON + 中文业务解释

背景：我负责物料主数据维护，需要理解表结构以便进行数据分析。
```

**价值分析**:
- ✅ 明确指定字段，避免获取无用数据
- ✅ 要求业务解释，帮助理解
- ✅ 指定输出格式，便于使用
- ✅ 说明业务背景，帮助 AI 理解需求

**预期收益**:
- 业务人员自助获取数据，IT 响应时间从 2 天 → 10 分钟
- 数据准确性提升（减少沟通误解）

### 场景 2: 业务流程诊断

**业务痛点**: 业务流程执行异常，需要快速定位问题根源

**价值**: 快速定位问题，减少业务中断时间

#### ❌ 低质量提示词
```
检查销售订单为什么变慢了
```

#### ✅ 高质量提示词模板

```markdown
我需要诊断 SAP 销售订单处理变慢的问题，请帮我：

1. **获取 VBFA 表（销售凭证流）结构**，识别关键字段
2. **获取前 20 条数据**，分析数据量和趋势
3. **查找相关的索引和表**：
   - VBAK (销售凭证头数据)
   - VBAP (销售凭证行项目)
4. **分析性能瓶颈**：哪些查询可能导致性能问题？
5. **建议优化方案**：索引优化、查询优化等

背景：销售订单创建和查询响应时间从 1 秒增加到 10 秒，影响用户体验。
```

**价值分析**:
- ✅ 指定具体的问题现象（响应时间）
- ✅ 提供诊断思路（表结构 → 数据 → 索引）
- ✅ 要求具体优化建议
- ✅ 说明业务影响

**预期收益**:
- 问题定位时间从 4 小时 → 30 分钟
- 提供可执行的优化建议

### 场景 3: 定制化报表开发

**业务痛点**: 标准报表无法满足业务需求，需要定制开发

**价值**: 快速生成定制报表，支持业务决策

#### ❌ 低质量提示词
```
生成一个报表
```

#### ✅ 高质量提示词模板

```markdown
我需要创建一个库存分析报表，请帮我：

1. **数据源**：
   - MSEG: 物料凭证
   - MKPF: 物料凭证头
   - MARA: 物料主数据
   - MARD: 物料库存

2. **需求字段**：
   - 物料号 (MATNR)
   - 物料描述 (MAKTX)
   - 工厂 (WERKS)
   - 库存地点 (LGORT)
   - 库存数量 (MENGE)
   - 入库日期 (BUDAT)

3. **筛选条件**：
   - 最近 30 天
   - 库存 > 0

4. **输出**：
   - Python FastAPI 接口代码
   - SQL 查询语句
   - Swagger 文档

5. **要求**：
   - 使用 ADT REST API 获取数据
   - 添加缓存机制
   - 错误处理和日志
```

**价值分析**:
- ✅ 明确数据源和表关系
- ✅ 指定具体字段
- ✅ 定义筛选条件
- ✅ 要求完整的技术实现

**预期收益**:
- 报表开发时间从 3 天 → 2 小时
- 一次开发，多次复用

### 场景 4: 接口对接支持

**业务痛点**: 需要将 SAP 数据对接至其他系统

**价值**: 标准化接口，降低集成成本

#### ❌ 低质量提示词
```
创建一个 API 获取 SAP 数据
```

#### ✅ 高质量提示词模板

```markdown
我需要创建一个 REST API，将 SAP 物料主数据同步到外围系统，请帮我：

1. **接口设计**：
   - GET /api/materials - 获取物料列表
   - GET /api/materials/{matnr} - 获取单个物料详情
   - GET /api/materials/sync - 触发全量同步

2. **数据映射**：
   | SAP 字段 | API 字段 | 说明 |
   |----------|----------|------|
   | MATNR | material_no | 物料号 |
   | MAKTX | material_desc | 物料描述 |
   | MTART | material_type | 物料类型 |

3. **技术要求**：
   - Python FastAPI 实现
   - 使用 SAP ADT REST API
   - JWT 认证
   - 响应格式：{success, data, message}
   - 限流保护

4. **文档输出**：
   - API 文档 (Swagger)
   - 接口说明文档
```

**价值分析**:
- ✅ 完整接口设计（路径、方法、参数）
- ✅ 数据映射表
- ✅ 技术要求明确
- ✅ 文档要求

**预期收益**:
- 接口开发标准化
- 减少沟通成本
- 便于维护和扩展

   - 潜在问题

输出格式：Markdown 文档，包含代码片段和中文解释

**价值分析**:
- ✅ 要求获取完整源码

- ✅ 风险评估

**预期收益**:
- 代码理解时间从 2 天 → 2 小时

## 提示词结构最佳实践

### 高价值提示词公式

```
# 公式：背景 + 目标 + 约束 + 格式 + 理由

[背景]：当前业务场景和痛点
[目标]：具体要达成的结果
[约束]：技术限制和条件
[格式]：期望的输出格式
[理由]：为什么这样做有价值
```

### 示例应用

```markdown
# 完整示例

【背景】
我是 SAP 物料管理模块的业务负责人，需要定期分析物料库存数据。

【目标】
创建自动化的库存报表接口，支持每日库存数据查询和导出。

【约束】
- 使用 Python FastAPI
- 通过 ADT REST API 获取数据
- 需要 JWT 认证
- 数据量预计 50 万+

【格式】
- Python 代码 + 配置文件
- API 文档 (Swagger)
- 使用说明

【理由】
目前每月手工导数据需要 2 人/天，希望自动化后释放人力用于数据分析工作
```

## 理由说明

### 为什么业务导向的提示词更有价值？

1. **明确业务目标**
   - 技术方案服务于业务目标
   - 避免过度设计或不足设计

2. **量化预期收益**
   - 设定明确的改进指标
   - 便于评估投入产出比

3. **促进跨部门协作**
   - 业务人员也能参与需求定义
   - 减少沟通损耗

### 为什么需要完整的上下文？

1. **精准匹配**
   - AI 理解真实需求
   - 避免方向偏差

2. **质量保证**
   - 输出更符合业务要求
   - 减少返工

3. **知识沉淀**
   - 文档化的需求更有价值
   - 便于复盘和传承
