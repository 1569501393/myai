## 一、认证与初始化

### 1.1 获取 CSRF Token

所有 POST/PUT/DELETE 请求都需要先获取 CSRF Token。

```bash
# 获取 CSRF Token 和 Cookie
curl -c cookies.txt -u "U1170:hysoft888999" \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/discovery" \
  -H "x-csrf-token: Fetch" -D headers.txt -k > /dev/null

# 提取 CSRF Token
CSRF=$(grep -i "^x-csrf-token" headers.txt | cut -d' ' -f2 | tr -d '\r')
```

### 1.2 认证参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 用户名 | `U1170` | SAP 用户 |
| 密码 | `hysoft888999` | SAP 密码 |
| 客户端 | `400` | SAP Client |

## 十六、完整 curl 示例脚本

```bash
#!/bin/bash
# SAP ADT REST API 测试脚本

# 配置
HOST="mysap.goodsap.cn"
PORT="50400"
USER="U1170"
PASS="hysoft888999"
BASE="http://${HOST}:${PORT}"

# Step 1: 获取 CSRF Token
echo "=== Step 1: 获取 CSRF Token ==="
curl -c cookies.txt -u "${USER}:${PASS}" \
  "${BASE}/sap/bc/adt/discovery" \
  -H "x-csrf-token: Fetch" -D headers.txt -k > /dev/null
CSRF=$(grep -i "^x-csrf-token" headers.txt | cut -d' ' -f2 | tr -d '\r')
echo "CSRF: $CSRF"
```

## 十九、常见问题

### 19.1 401 未授权

检查用户名密码是否正确，或 Session 是否过期。

### 19.2 404 资源不存在

确认对象名称大小写正确，SAP 对象名通常区分大小写。

### 19.3 参数未找到

检查 URL 参数格式，ADT API 通常使用路径参数而非查询参数。

### 19.4 CSRF Token 无效

每次会话只需获取一次 Token，但长时间操作后可能需要刷新。
