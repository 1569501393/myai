## 五、安全风险评估

### 5.1 高风险项

| 风险项 | 风险等级 | 说明 |
|--------|----------|------|
| 角色数量过多 | 🔴 高 | 200 个角色可能导致权限过大 |
| 开发角色 | 🔴 高 | 拥有 SEBS/SES 管理员权限 |
| 财务角色 | 🔴 高 | 拥有多个财务模块权限 |
| 管理员角色 | 🔴 高 | SAP_BR_ADMINISTRATOR |

### 5.2 建议措施

| 建议 | 优先级 | 说明 |
|------|--------|------|
| 权限分离 | 高 | 建议将开发和业务权限分离 |
| 定期审计 | 高 | 定期审查用户权限 |
| 最小权限 | 中 | 遵循最小权限原则 |
| 敏感操作审计 | 高 | 记录敏感操作的日志 |

## 八、curl 命令示例

### 8.1 获取 CSRF Token

```bash
curl -c cookies.txt -u "U1170:hysoft888999" \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/discovery" \
  -H "x-csrf-token: Fetch" -D headers.txt -k > /dev/null

CSRF=$(grep -i "^x-csrf-token" headers.txt | cut -d' ' -f2 | tr -d '\r')
```

### 8.2 查询用户角色

```bash
curl -b cookies.txt \
  "http://mysap.goodsap.cn:50400/sap/bc/adt/datapreview/freestyle?rowNumber=200" \
```
