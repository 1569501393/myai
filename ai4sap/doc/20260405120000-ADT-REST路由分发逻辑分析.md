## 三、核心方法详解

### 3.1 GET_REQUEST_PROXY

**代码位置**: 行 295-337

```abap
METHOD get_request_proxy.
  DATA:
    uri          TYPE string,
    path         TYPE string,
    system_alias TYPE if_gw_service=>gty_system_alias,
    destination  TYPE rfcdest.

  " Step 1: 系统别名检查
  system_alias_checkpoint( EXPORTING p_server = p_server
                          IMPORTING p_uri   = uri
                                    p_path  = path
                                    p_system_alias = system_alias ).

  " 如果有系统别名 → RFC FES
  IF system_alias IS NOT INITIAL.
    service_authority_check( ).
    destination = get_destination( system_alias ).
    IF destination IS NOT INITIAL.
      result = get_rfc_request_proxy_fes( ... ).
      RETURN.
    ELSE.
      " 移除 URI 中的别名
      remove_alias_from_uri( p_server = p_server 
                             p_system_alias = system_alias ).
    ENDIF.
  ENDIF.

  " Step 2: 调试器检查
  destination = get_debugger_destination( p_server ).
  IF destination IS NOT INITIAL.
    result = get_rfc_request_proxy_dbg( ... ).
    RETURN.
  ENDIF.

  " Step 3: 追踪检查
  IF p_server->request->get_header_field( name = co_header_adt_trace ) IS NOT INITIAL.
    result = get_rfc_request_proxy_trc( p_server ).
    RETURN.
  ENDIF.

  " Step 4: 默认 HTTP 代理
  result = get_http_request_proxy( p_server ).
ENDMETHOD.
```

### 3.2 SYSTEM_ALIAS_CHECKPOINT

**代码位置**: 行 473-533

**功能**: 解析 URI 中的系统别名信息

```abap
METHOD system_alias_checkpoint.
  " 获取 URI 和 PATH
  p_path = p_server->request->get_header_field( name = if_http_header_fields_sap=>path ).
  p_uri  = p_server->request->get_header_field( name = if_http_header_fields_sap=>request_uri ).

  " 获取 Gateway 配置的系统别名
  get_system_aliases_configured( IMPORTING e_system_aliases = adt_aliases ).

  " 使用正则表达式查找别名: /sap/bc/adt;o=ALIAS_NAME/
  FIND REGEX |/sap/bc/adt;o=([A-Z_0-9]+)/| IN p_uri SUBMATCHES alias_string.

  IF sy-subrc = 0.  " 显式路由
    p_system_alias = alias_string.
    " 验证别名在 Gateway 中配置
    READ TABLE adt_aliases TRANSPORTING NO FIELDS
      WITH KEY system_alias = p_system_alias.
    IF sy-subrc <> 0.
      RAISE EXCEPTION TYPE cx_adt_rest_routing.
    ENDIF.
    " 从 URI 中移除别名标记
    replace_alias_from_uri( ... ).
  ELSE.  " 隐式路由
    IF lines( adt_aliases ) = 0.
      " 无路由: 本地处理
      RETURN.
    ELSEIF lines( adt_aliases ) = 1.
      " 单一别名: 使用该别名
      p_system_alias = adt_aliases[1].
    ELSE.
      " 多别名: 查找默认别名
      LOOP AT adt_aliases INTO adt_alias WHERE is_default = abap_true.
        IF p_system_alias IS INITIAL.
          p_system_alias = adt_alias-system_alias.
        ELSE.
          " 多个默认别名: 清除路由
          CLEAR p_system_alias.
          EXIT.
        ENDIF.
      ENDLOOP.
    ENDIF.
  ENDIF.
ENDMETHOD.
```

### 3.3 GET_DEBUGGER_DESTINATION

**代码位置**: 行 240-262

```abap
METHOD get_debugger_destination.
  " 获取目标服务器头
  data(target_server) = p_server->request->get_header_field(
    if_adt_header=>co_header_server_instance ).

  " 检查是否非本机
  IF target_server IS NOT INITIAL AND
     target_server NE cl_abap_syst=>get_instance_name( ).

    " 获取服务器列表
    CALL FUNCTION 'TH_SERVER_LIST'
      TABLES list = servers.

    " 检查目标服务器是否在白名单中
    READ TABLE servers TRANSPORTING NO FIELDS
      WITH KEY name = target_server.
    IF sy-subrc = 0.
      result = target_server.  " 路由到其他实例
    ENDIF.
  ENDIF.
ENDMETHOD.
```

### 3.4 GET_DESTINATION

**代码位置**: 行 265-281

```abap
METHOD get_destination.
  CALL FUNCTION 'RFC_DESTINATION_FINDER'
    EXPORTING i_system_alias = p_system_alias
    IMPORTING e_rfc_destination = result.

  IF result IS INITIAL.
    RAISE EXCEPTION TYPE cx_adt_rest_routing.
  ELSEIF result = co_rfc_destination_none.
    CLEAR result.  " NONE 表示本地处理
  ENDIF.
ENDMETHOD.
```

## 四、URI 格式规范

### 4.1 标准 URI 格式

```
/sap/bc/adt/{resource_path}
```

### 4.2 带系统别名的 URI

```
/sap/bc/adt;o={SYSTEM_ALIAS}/{resource_path}
```

**示例**:
```
/sap/bc/adt;o=S4H/oo/classes/ZCL_EXAMPLE/source/main
```

## 五、HTTP 头字段

| 头字段 | 常量定义 | 说明 |
|--------|----------|------|
| `X-sap-adt-sessiontype` | `co_session_type` | 会话类型 (stateful/stateless) |
| `sap-adt-saplb` | `co_header_adt_saplb` | SAP Load Balancer 信息 |
| `X-adt-runtime-tracing` | `co_header_adt_trace` | 运行时追踪 |
| `X-sap-adt-server-instance` | `if_adt_header=>co_header_server_instance` | 目标服务器实例 |
| `X-csrf-token` | `if_rest_request=>gc_header_csrf_token` | CSRF 令牌 |
| `sap-cache-control` | - | SAP 缓存控制 |
| `cache-control` | - | 标准缓存控制 |

## 七、代理实现类

| 代理类 | 用途 | 处理方式 |
|--------|------|----------|
| `LCL_HTTP_REQUEST_PROXY` | 本地处理 | ICF 直接处理 |
| `LCL_RFC_REQUEST_PROXY_FES` | 前端服务路由 | RFC 到目标系统 |
| `LCL_RFC_REQUEST_PROXY_DBG` | 调试器重定向 | RFC 到调试服务器 |
| `LCL_RFC_REQUEST_PROXY_TRC` | 追踪请求 | RFC 到追踪服务 |

## 九、总结

### 路由决策矩阵

| 条件 | 代理类型 | 说明 |
|------|----------|------|
| URI 包含 `;o=ALIAS` 且 RFC 目的地存在 | RFC FES | 路由到远程系统 |
| `X-sap-adt-server-instance` 指向其他实例 | RFC DBG | 路由到调试服务器 |
| `X-adt-runtime-tracing` 头存在 | RFC TRC | 路由到追踪服务 |
| 其他情况 | HTTP Proxy | 本地 ICF 处理 |

### 关键常量

```abap
co_service_path    = '/sap/bc/adt'       " 服务路径
co_service_name   = 'ADT_0001'         " 服务名称
co_session_type    = 'X-sap-adt-sessiontype'  " 会话头
co_header_adt_saplb = 'sap-adt-saplb'  " SAP LB 头
co_header_adt_trace = 'X-adt-runtime-tracing' " 追踪头
```
