# 测试新端点获取 SAP 表结构和数据

## 测试背景
根据用户提供的新 API 路径：
- 表结构：`/sap/bc/adt/dictionary/tables/{表名}/definition`
- 表数据：`/sap/bc/adt/dictionary/tabledata/{表名}?rows=10`

我们对 MARA 和 ZTJQ0001 两张表进行了测试。

## 测试环境
- SAP 系统：http://mysap.goodsap.cn:50400
- 客户端：400
- 用户：U1170
- 认证：Basic Auth

## 测试结果

### MARA 表
| 端点 | 方法 | 参数 | 状态码 | 说明 |
|------|------|------|--------|------|
| `/sap/bc/adt/dictionary/tables/MARA/definition` | GET | 无 | 404 | 资源未找到 |
| `/sap/bc/adt/dictionary/tabledata/MARA?rows=10` | GET | rows=10 | 404 | 资源未找到 |

### ZTJQ0001 表
| 端点 | 方法 | 参数 | 状态码 | 说明 |
|------|------|------|--------|------|
| `/sap/bc/adt/dictionary/tables/ZTJQ0001/definition` | GET | 无 | 404 | 资源未找到 |
| `/sap/bc/adt/dictionary/tabledata/ZTJQ0001?rows=10` | GET | rows=10 | 404 | 资源未找到 |

## 分析
1. 这两个端点在当前 SAP 系统中不可用（均返回 404）。
2. 这可能由于以下原因：
   - 这些端点在当前 SAP 基线版本中不存在或未实现。
   - 这些端点需要特定的补丁级别或增强包才能可用。
   - 这些端点可能被误写，实际路径不同。
3. 相比之下，我们之前验证通过的端点是：
   - 表结构：`/sap/bc/adt/ddic/tables/{表名}/source/main`（获取 DDLS 源码）
   - 表数据：`/sap/bc/rest/odi/v3/{表名}`、`/sap/bc/rest/odi/{表名}`、`/sap/bc/adt/ddic/tables/{表名}/data`（但这些在当前系统中也不启用，返回 404 或 403）

## 结论
- 所提供的新端点 `/sap/bc/adt/dictionary/tables/*/definition` 和 `/sap/bc/adt/dictionary/tabledata/*?rows=10` 在目标 SAP 系统中不可用。
- 获取表结构仍需使用 `/sap/bc/adt/ddic/tables/{表名}/source/main`（返回 DDLS 源码，需自行解析）。
- 获取表数据在当前系统中需要通过其他方式（如自定义 OData 服务、BAPI、函数模块或直接 SQL，因为标准 OData/ADT 表数据端点未启用）。

## 建议
1. 如果需要使用这些特定端点，请咨询 SAP 管理员确认它们是否在其他系统或更高补丁级别中可用。
2. 继续使用已知的结构获取端点，并结合解析 DDLS 获取字段信息。
3. 对于表数据，评估是否可以在网关上启用标准 OData 服务，或创建自定义服务。
