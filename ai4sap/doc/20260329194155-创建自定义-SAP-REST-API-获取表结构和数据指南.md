# 如何创建自定义 SAP REST API 来获取表结构和数据

## 概述

虽然 SAP 提供了标准的 ADT 和 OData 端点，但在许多情况下，创建自定义 REST API 能更好地满足特定业务需求，比如：
- 需要联合多个表的数据
- 需要执行复杂的业务逻辑或过滤
- 需要返回特定格式的JSON（不是标准OData）
- 需要对访问进行更细粒度的控制
- 标准端点由于权限或配置原因不可用

本指南将展示两种主要方法来创建自定义 SAP REST API：
1. 使用 **SAP Gateway (SEGW)** 创建 OData 服务（推荐，适用于多数场景）
2. 使用 **ABAP RESTful Application Programming Model (RAP)**（现代方法，适用于新开发）
3. 使用经典的 **HTTP 服务 handler**（通过 `IF_HTTP_EXTENSION`，适用于高度定制）

我们将重点介绍第一种方法（SAP Gateway），因为它是最成熟、最易用且适合表数据访问的方法。

## 方法一：使用 SAP Gateway (SEGW) 创建 OData 服务

### 步骤概览
1. 定义数据模型（Entity Type, Entity Set，基于 DDIC 表或 CDS 视图）
2. 实现服务方法（GET_ENTITYSET, GET_ENTITY, 等）
3. 注册和激活服务
4. 测试文档和访问

### 详细步骤

#### 步骤 1: 创建项目和数据模型
- 事务码：`SEGW`
- 项目名称：`ZTABLE_API_DEMO`
- 描述：`Demo API for Table Structure and Data`
- 选择项目类型：**底层对象**（如果基于 DDIC 表或视图）或 **类型**。

#### 步骤 2: 导入 DDIC 表或 CDS 视图作为数据源
- 右键项目 → **导入** → **DDIC 结构**
- 输入表名，例如 `MARA`（物料主数据）或自定义表如 `ZTJQ0001`
- 系统将自动生成实体类型及其属性。
- 或者，导入 **CDS 视图**（如果你希望先定义一个视图来封装业务逻辑）：
  - 在 SEGW 中选择 **导入** → **CDS 视图**
  - 输入 CDS 视图名（例如：`ZCDS_MARA_BASIC`）

#### 步骤 3: 生成并调整服务实现
- 右键实体集 → **生成** → **服务实现**
- 系统会生成一个类（如 `/IWBEP/CL_MGW_ABS_FACTORY` 的子类）和方法。
- 打开生成的类（通常是 `ZCL_XXXXXXXX_DPC_EXT`），你可以在其中重写方法：
  - `GET_ENTITYSET`：用于获取多条记录（相当于 SELECT *)
  - `GET_ENTITY`：用于获取单条记录（由 KEY 决定）
  - `CREATE_ENTITY`, `UPDATE_ENTITY`, `DELETE_ENTITY`（如果需要）

#### 示例：重写 GET_ENTITYSET 以添加自定义过滤或排序
```abap
METHOD /iwbep/if_mgw_appl_srv_rgt~get_entityset.
  DATA: lt_mara TYPE TABLE OF mara,
        ls_mara LIKE LINE OF lt_mara.
  DATA: lt_entityset TYPE /iwbep/t_mgw_tech_data,
        ls_entityset LIKE LINE OF lt_entityset.

  " 可选：读取查询选项（如 $filter, $top, $skip, $orderby）
  DATA(lo_tech_params) = io_tech_request_context->get_tech_params( ).

  " 读取表数据（这里演示全表读取，实际应加入 WHERE 条件以避免全表扫描）
  SELECT * FROM mara INTO TABLE @lt_mara UP TO 100 ROWS. " 示例：限制 100 条，实际应尊重 $top

  LOOP AT lt_mara INTO ls_mara.
    " 将 MARA 记录映射为实体属性
    ls_entityset-value = ls_mara-matnr.
    ls_entityset-name = 'Matnr'.
    APPEND ls_entityset TO lt_entityset.
    CLEAR: ls_entityset.

    ls_entityset-value = ls_mara-mtart.
    ls_entityset-name = 'Mtart'.
    APPEND ls_entityset TO lt_entityset.
    CLEAR: ls_entityset.

    " 继续其他字段...
    " 为了简化，这里只演示两个字段。实际应遍历所有需要暴露的字段。
  ENDLOOP.

  " 设置响应数据
  er_entityset = lt_entityset.
ENDMETHOD.
```

> 注：上面是简化示例。真实实现中，你应使用 `/IWBEP/IF_MGW_APPL_SRV_RGT~CREATE_ENTITIES` 方法或使用模型提供的辅助类来正确构建复杂结构。

#### 步骤 4: 注册和激活服务
- 在 SEGW 中，右键项目 → **生成** → **对象**
- 然后右键项目 → **生成** → **服务**
- 生成后，转到事务码：`/IWFND/MAINT_SERVICE`
- 输入服务名（例如：`ZTABLE_API_DEMO_SRV`），检查是否已注册。
- 如果未注册，点击 **添加服务**，选择你的服务，输入系统别名（通常是 `LOCAL`），然后 **添加**。
- 激活服务。

#### 步骤 5: 测试服务
- 使用浏览器或工具（如 Postman）访问：
  ```
  http://mysap.goodsap.cn:50400/sap/opu/odata/sap/ZTABLE_API_DEMO_SRV/ZtmaraCollection?$format=json&$top=3
  ```
  （注：实体集名称取决于你在 SEGW 中的命名，通常是表名加 Collection）

- 你也可以在网关浏览器中测试：`/IWFND/GROOT_VIEWER`

### 方法二：使用 ABAP RESTful Application Programming Model (RAP)（云优先，适用于 S/4BW 和 ABAP 平台）
#### 简要步骤
1. 定义 CDS 视图作为数据模型（带 `@OData.publish: true`）
2. 定义服务定义（`.psdl` 文件）
3. 实现服务绑定（如果需要自定义逻辑）
4. 生成对象并激活
5. 服务自动发布在 `/sap/opu/odata/sap/<服务名>/`

#### 示例 CDS 视图（带 OData 注解）
```abap
@EndUserText.label: 'Material Basic Info'
@OData.publish: true
define view ZcdsMaterialBasic as select from mara {
    key matnr,
    mtart as MaterialType,
    mbrsh as IndustrySector,
    matkl as MaterialGroup,
    ersda as CreatedOn,
    erzet as CreatedAt,
    ernam as CreatedBy,
    aenam as ChangedBy,
    laeda as ChangedOn
}
```

#### 服务定义（ZI_MATERIAL_BASIC.psdl）
```psdl
service {
    expose ZcdsMaterialBasic as MaterialBasic;
}
```

#### 激活后访问
- 元数据：`/sap/opu/odata/sap/ZCDS_MATERIAL_BASIC_SRV/$metadata`
- 实体集：`/sap/opu/odata/sap/ZCDS_MATERIAL_BASIC_SRV/MaterialBasic?$top=3`

### 方法三：使用经典 HTTP 服务 handler（IF_HTTP_EXTENSION）
适用于需要完全控制响应格式（例如返回非标准 JSON、XML 或 HTML）的场景。

#### 步骤
1. 创建一个 Z类，实现接口 `IF_HTTP_EXTENSION`
2. 在事务 `SICF` 下创建一个自定义节点（例如 `/sap/z_rest/table_data`）
3. 将该节点的处理程序指向你的 Z类
4. 在类中实现 `HANDLE_REQUEST` 方法，读取表数据并返回自定义响应

#### 示例：返回简化的 JSON 数组
```abap
CLASS zcl_table_rest_handler DEFINITION
  PUBLIC
  INHERITING FROM cl_http_handler.
  PUBLIC SECTION.
    METHODS:
      if_http_extension~handle_request.
ENDCLASS.

CLASS zcl_table_rest_handler IMPLEMENTATION.
  METHOD if_http_extension~handle_request.
    DATA: lt_mara TYPE TABLE OF mara,
          lv_json TYPE string.

    SELECT matnr mtart ersda FROM mara INTO TABLE @lt_mara UP TO 10 ROWS.

    " 将内表转换为简单 JSON（此处为示例，实际应使用 /ui2/cl_json 或写自定义循环）
    TRY.
        lv_json = /ui2/cl_json=>serialize( data = lt_mara ).
      CATCH cx_sy_move_cast_error.
    ENDTRY.

    server->response->set_content_type( 'application/json' ).
    server->response->set_cdata( lv_json ).
    server->response->set_status( code = 200 reason = 'Ok' ).
  ENDMETHOD.
ENDCLASS.
```

然后在 SICF 中创建节点：
- 节点：`/sap/z_rest/table_data`
- 处理程序：类名 `ZCL_TABLE_REST_HANDLER`

访问：`http://mysap.goodsap.cn:50400/sap/z_rest/table_data`

## 最佳实践

| 实践 | 说明 |
|------|------|
| **优先使用 CDS + OData / RAP** | 现代、声明式、安全、性能好，适合云原生和 S/4HANA |
| **最小化数据暴露** | 只暴露必要字段，避免返回敏感字段（如工资、密码等） |
| **使用分页和过滤** | 永远尊重 `$top`, `$skip`, `$filter`, `$orderby`，防止全表扫描 |
| **输入验证** | 虽然 SAP 框架会处理很多注入风险，但自定义逻辑仍需注意防止 SQL 注入（如使用参数查询） |
| **使用 HTTPS** | 生产环境必须使用 HTTPS 保护凭据和数据 |
| **授权最小化** | 为服务分配最小必需的授权对象（如 `S_SERVICE`，`S_RFCACL`，以及后台对象的授权） |
| **监控和日志** | 使用 `/IWFND/ERROR_LOG`, `SMICM`, `ST22` 监控异常和性能 |
| **版本控制** | 对服务进行版本管理（如 V1、V2），避免破坏现有客户端 |
| **文档化** | 提供清晰的 API 文档（端点、方法、参数、响应码、示例），可使用 Swagger 或 SAP API Hub |
| **错误处理** | 返回适当的 HTTP 状态码（400、401、403、404、500）以及有意义的错误信息 |
| **避免直接表访问（如果可能）** | 除非绝对必要，否则尽量使用 BAPI、函数模块或 CDS 视图，以确保业务逻辑和一致性 |
| **考虑使用 SAP API Management** | 为额外的安全、速率限制、分析和门户功能添加 API 管理层 |

## 结论

创建自定义 SAP REST API 来获取表结构和数据是一个强大的能力，使你能够将 SAP 集成到现代应用程序中。通过遵循上述步骤和最佳实践，你可以构建安全、高效且易于维护的服务。

推荐的首选方案是：**CDS 视图 + OData 服务（使用 SEGW 或 RAP）**，因为它结合了现代开发优势、安全性和易用性。

生成的服务可以被任何支持 HTTP 的客户端消费，包括移动应用、Web 应用、微服务和报告工具。
